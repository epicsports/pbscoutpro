## 🐛 i18n call-SHAPE crash class (2026-06-14) — FIXED + lint shipped

**SIBLING class to the t-scope one above, distinct mechanism (Jacek-reported prod crash: AdminPlayersPage "Strona undefined z undefined" / `t('admin_teams_page_n_of_m') is not a function`).** Root cause: `useLanguage`'s `t()` **auto-invokes** function values — `t(key, ...args) => typeof v==='function' ? v(...args) : v`. So the ONLY correct way to pass interpolation args is `t('k', a, b)`. The footgun `t('k')(a, b)` invokes the function with NO args first (→ a string like "Strona undefined z undefined"), then calls that string → crash. **10 call sites carried the footgun** (genuine current-source bugs, NOT stale cache — that earlier hypothesis was wrong): AdminPlayers + AdminTeams pagers, all 4 MergePlayersModal interpolators (would crash the merge-players flow on open), `ppt_step_counter` (also a MISSING key), `reset_sent`, `line_draw_banner`, `user_global_role_promote/revoke_body` (the last 4 were ALSO static strings with `{placeholder}` → double bug). All fixed to `t('k', …)` + the string keys converted to function interpolators.
> **Why it escaped green:** the pagers only render at `totalPages > 1` — the 15-player e2e seed never paginated (twin of the Quick-log coverage gap). Seed now carries 40 padding players (55 global → 2 pages); `tests/e2e/admin-pagination.spec.js` exercises the pager (fail-first: it was RED on the buggy code, GREEN after).
> **✅ Tooling gap CLOSED:** `scripts/lint-i18n-shapes.js` (wired into `npm run precommit`) — Rule A flags `t('k')(…)` chained invocation (the crash footgun); Rule B flags `t('k', …args)` where `k` isn't a function in both langs (args silently dropped → literal `{placeholder}`).
